# Memoria operativa local

## Propósito

Este archivo funciona como respaldo local de memoria para agentes cuando no haya persistencia externa disponible. Resume lo mínimo que un agente debe recordar para no entrar ciego.

## Estado del proyecto

- Repo creado para jugar Chile con memoria persistente en Markdown.
- La estructura está pensada como vault híbrido estilo Obsidian + docs operativos.
- El repo debe servir como fuente de verdad para turnos, estrategia, diplomacia y contexto histórico.

## Prioridad estratégica vigente

Consolidar:

- corredor bioceánico
- microelectrónica soberana
- certificación transpacífica con Japón y Corea
- estabilidad contractual y de soporte en India
- cierre técnico de Reforma 2017

## Riesgo principal

Sobreextensión. El usuario quiere conducción seria de Estado, no expansión vacía.

## Cómo trabajar en este repo

- leer primero `README.md`
- después `estado/estado-actual-2016-09-03.md`
- luego `pendientes/pila-estrategica.md`
- crear nuevos archivos de turno cuando corresponda
- actualizar pendientes y estado cuando cambie la situación

## Convención de actualización

Cada avance importante debería reflejarse en:

1. `turnos/`
2. `estado/`
3. `pendientes/`
4. `README.md` si cambia la prioridad central
